# Phase 11: Testing & CI/CD

## Topics

- Workflow testing strategy
- Mock APIs for deterministic tests
- Git-based workflow versioning
- CI/CD pipelines for n8n
- Infrastructure as Code (Terraform)

## Hands-on

1. Store workflows in GitHub
2. Automate deployment via CI pipeline

## CI/CD Stages

1. Validate workflow JSON schema
2. Run integration tests against mock APIs
3. Deploy to staging n8n
4. Promote to production with approvals

## Success Criteria

- Every workflow change is reviewable and traceable
- Deployment is automated and repeatable
- Rollbacks are documented and tested
